Project 1 — Parametric flange or end cap

Goal: master Part Design + spreadsheet-driven parameters. Build a bolt-hole flange where bolt count, bolt-circle diameter, and thickness are all driven from a spreadsheet, so changing one value rebuilds the whole part cleanly.

Skills shown: sketching, constraints, parametric tree, design intent.

Reference: MangoJelly's "FreeCAD spreadsheet" tutorials, and the official Part Design tutorial in the FreeCAD wiki.
Project 2 — Sheet-metal bracket

Goal: learn the Sheet Metal workbench. Make an L- or U-bracket with proper bend radii, relief cuts, and an unfolded flat pattern (which is what a manufacturer actually cuts).

Skills shown: manufacturability awareness, bends, flat patterns.

Reference: search "FreeCAD sheet metal workbench tutorial" — MangoJelly and the workbench's own wiki page both cover this well.
Project 3 — Reverse-engineered real part

Goal: measure a real object (a caliper, a hinge, a hose fitting) with a ruler/caliper and model it accurately. This is the most relatable skill to a hiring manager because it mirrors real shop work.

Skills shown: measurement, tolerance thinking, working from physical objects.

Reference: GrabCAD for comparison models; any caliper-measurement tutorial.
Project 4 — Multi-part assembly

Goal: combine 3–5 parts into a working assembly with mating constraints. A small bench vise, a caster wheel, or a two-stage gear pair are all strong choices. Use the Assembly workbench (built into FreeCAD 1.0+).

Skills shown: assemblies, mates, interference checking, exploded views.

Reference: MangoJelly's FreeCAD 1.0 assembly series — they have a dedicated playlist for the new built-in assembly tools.
Project 5 — Full technical drawing of the assembly

Goal: take Project 4 into the TechDraw workbench and produce a proper engineering drawing: orthographic views, section view, exploded isometric, full dimensions, tolerances, a bill of materials, and a title block.

Skills shown: GD&T basics, drawing standards, communication — this is the piece that most separates a hobbyist portfolio from a professional one.

Reference: MangoJelly's TechDraw tutorials; the TechDraw wiki page for the workbench mechanics.
A few notes on sequencing. Projects 1–3 are standalone parts you can knock out fairly quickly. Project 4 should reuse good habits from the earlier ones (spreadsheet parameters, clean sketches), and Project 5 depends entirely on Project 4 existing. If you only have time for a lean portfolio, Projects 3, 4, and 5 together tell the strongest story: "I can measure something real, model it as a working assembly, and produce manufacturing-ready drawings."
